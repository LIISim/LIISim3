Custom Import (MSExcel: Unicode .txt)

The custom file import scans a directory for files with the extension '*.txt', '*.dat' or '*.csv', with the filename matching the pattern preset. The data delimiter, decimal mark and time unit can be choosen by the user.

1) Data format

The signal data can be either arranged in one file in multiple columns or spread over multiple files. If you like to use the single file version, the data should be composed as described below:

- first column: time data
- second column: signal data (Channel 1)
- third column: signal data (Channel 2)
- fourth column: signal data (Channel 3)
- ...

Example:
Time Channel 1 Channel 2 ...
6e-009 0.0202528 0.0212294
8e-009 0.0196180 0.0212294
1e-008 0.0208387 0.0212294
1.2e-008 0.0195692 0.0212294
...


If you would like the multi-file version, you should select the 'Only one channel per file'-checkbox. The filename should contain a number identifying the channel and the file name pattern the <CHANNEL>, otherwise this will result in an error. The data inside the file should be composed as follows:

- first column: time data
- second column: signal data

Example:
Time Signal Data
6e-009 0.0202528
8e-009 0.0196180
1e-008 0.0208387
1.2e-008 0.0195692
...


A series of signals can be imported using the <SIGNAL> identifier. The filename should in this case contain a ascending number used to define the signal number, e.g:
signal_0001.txt

2) Filename Example

Filename: FirstTest_1_01_ABC.txt

Filename pattern: <RUN> _ <CHANNEL> _ <SIGNAL> _ABC .txt
Element in filename Corresponding element in filename pattern Info
FirstTest <RUN> Will be used as name
_ _ Ignored
1 <CHANNEL> Channel identifier, if multiple files are used
_ _ Ignored
01 <SIGNAL> Signal series identifier
_ABC _ABC Ignored
.txt .txt

3) MRun Settings

Settings for the run are saved in a file named <RUN>_settings.txt. If this file is available for a run, it will be automatically loaded when a valid file (according to the pattern) is detected.
